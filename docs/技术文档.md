### 2. 情感分析模块

#### 分析方法
基于词典的情感分析，核心算法：

```
情感得分 = Σ(情感词权重 × 程度副词权重 × 否定词修正)
```

#### 词典结构
- **正面词典**: 包含积极情感词汇
- **负面词典**: 包含消极情感词汇
- **程度副词**: 修饰情感强度（如"非常"=2.0）
- **否定词**: 反转情感极性（如"不"）

#### 分类规则
- score > 0.1: 正面
- -0.1 ≤ score ≤ 0.1: 中性
- score < -0.1: 负面

#### 优势
1. **可解释性强**: 每个判断可追溯到具体词汇
2. **无需训练**: 不依赖标注数据
3. **领域适应**: 快速扩展新领域词典
4. **低成本**: 无需GPU等高端硬件

### 3. 话题分析模块

#### 关键词提取
使用TF-IDF算法：
```
TF-IDF(t,d) = TF(t,d) × IDF(t)
```

#### 话题聚类
- 算法: K-means
- 特征: TF-IDF向量
- 输出: 话题簇 + 关键词

### 4. 可视化模块

#### 图表类型
1. **情感分布饼图**: 展示正面/中性/负面比例
2. **情感趋势图**: 时间序列折线图
3. **热词云**: 词频可视化
4. **地理分布**: 舆情地域分布

#### 技术实现
- ECharts: 图表渲染
- Vue3: 响应式数据绑定

### 5. 预警模块

#### 预警类型
1. **负面激增**: 负面比例超过阈值
2. **热点话题**: 关键词频率异常
3. **关键词监控**: 敏感词出现

#### 触发机制
- 定时检查（每小时）

## 数据库设计

### articles表
```sql
CREATE TABLE articles (
    id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(500) NOT NULL,
    content TEXT NOT NULL,
    source VARCHAR(100),
    url VARCHAR(500) UNIQUE,
    author VARCHAR(100),
    publish_time DATETIME,
    crawl_time DATETIME,
    sentiment VARCHAR(20),
    sentiment_score FLOAT,
    keywords TEXT,
    topic VARCHAR(200),
    view_count INT DEFAULT 0,
    comment_count INT DEFAULT 0,
    share_count INT DEFAULT 0,
    INDEX idx_publish_time (publish_time),
    INDEX idx_sentiment (sentiment),
    INDEX idx_source (source)
);
```

### alerts表
```sql
CREATE TABLE alerts (
    id INT PRIMARY KEY AUTO_INCREMENT,
    alert_type VARCHAR(50),
    level VARCHAR(20),
    title VARCHAR(200),
    description TEXT,
    trigger_value FLOAT,
    threshold FLOAT,
    create_time DATETIME,
    status VARCHAR(20) DEFAULT 'pending',
    INDEX idx_create_time (create_time),
    INDEX idx_status (status)
);
```

## API接口

### 数据接口
- `GET /api/data/articles` - 获取文章列表
- `GET /api/data/articles/:id` - 获取文章详情

### 分析接口
- `POST /api/sentiment/analyze` - 分析文本情感
- `GET /api/analysis/sentiment-distribution` - 情感分布统计
- `GET /api/analysis/hotwords` - 获取热词
- `GET /api/analysis/trend` - 情感趋势

### 预警接口
- `GET /api/alert/list` - 获取预警列表
- `POST /api/alert/handle` - 处理预警
