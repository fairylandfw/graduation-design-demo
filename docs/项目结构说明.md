## 核心文件说明

### 后端核心文件

#### 1. `backend/app/__init__.py`
- 应用工厂函数
- 初始化数据库、Redis、JWT
- 注册蓝图路由

#### 2. `backend/analyzer/sentiment_analyzer.py`
- 基于词典的情感分析器
- 支持程度副词和否定词修正
- 返回情感类别和得分

#### 3. `backend/analyzer/topic_analyzer.py`
- TF-IDF关键词提取
- 词频统计热词发现
- K-means话题聚类

#### 4. `backend/app/api/__init__.py`
- RESTful API接口定义
- 包含数据、分析、预警等接口
- 支持分页、筛选、排序

#### 5. `backend/crawler/spiders/news_spider.py`
- Scrapy爬虫示例
- 可扩展支持多种数据源
- 自动去重和数据清洗

### 前端核心文件

#### 1. `frontend/src/main.js`
- Vue应用入口
- 注册路由和UI组件库
- 全局配置

#### 2. `frontend/src/views/Dashboard.vue`
- 仪表盘页面
- 集成多种图表展示
- 实时数据更新

#### 3. `frontend/src/api/index.js`
- Axios封装
- API接口统一管理
- 请求拦截和错误处理

### 数据文件

#### 1. `data/dictionaries/`
- **positive.txt**: 正面情感词汇库
- **negative.txt**: 负面情感词汇库
- **degree.txt**: 程度副词及权重
- **negation.txt**: 否定词列表

可根据具体领域扩展词典内容

#### 2. `data/stopwords/stopwords.txt`
- 中文停用词表
- 用于分词后过滤
- 提升关键词提取质量

## 模块依赖关系

```
前端 (Vue.js)
    ↓ HTTP请求
后端API (Flask)
    ↓ 调用
分析模块 (Analyzer)
    ↓ 读取
词典数据 (Dictionaries)

爬虫模块 (Crawler)
    ↓ 存储
数据库 (MySQL)
    ↑ 查询
后端API (Flask)
```

## 数据流向

```
1. 数据采集流程:
   网站 → 爬虫 → 数据清洗 → Redis去重 → MySQL存储

2. 分析流程:
   MySQL → 读取文本 → 分词 → 情感分析 → 更新数据库

3. 展示流程:
   前端请求 → 后端API → 数据库查询 → JSON响应 → 图表渲染
```

## 扩展指南

### 添加新爬虫
1. 在 `backend/crawler/spiders/` 创建新文件
2. 继承 `scrapy.Spider`
3. 实现 `parse` 方法

### 添加新API
1. 在 `backend/app/api/__init__.py` 添加路由
2. 实现处理函数
3. 返回JSON格式数据

### 添加新页面
1. 在 `frontend/src/views/` 创建Vue组件
2. 在 `main.js` 注册路由
3. 调用API获取数据

### 扩展词典
直接编辑 `data/dictionaries/` 下的文本文件，每行一个词

## 注意事项

1. **词典维护**: 定期更新情感词典以适应新词汇
2. **数据清理**: 定期清理历史数据避免数据库膨胀
3. **爬虫合规**: 遵守robots.txt和网站使用条款
4. **性能监控**: 关注数据库查询性能和API响应时间
5. **安全防护**: 修改默认密钥，启用HTTPS
