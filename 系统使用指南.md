## 🚀 启动前端

### 方式1：手动启动
打开新的命令行窗口：
```bash
cd C:\Users\Administrator\sentiment-analysis-platform\frontend
npm run serve
```

### 方式2：使用启动脚本
双击运行 `start.bat`（会自动启动前后端）

## 🌐 访问系统

## 🔄 更新数据

### 手动采集新数据
```bash
cd backend
venv\Scripts\activate
python auto_crawler.py
```

每次运行会采集12条新数据，自动去重。

## 📝 API接口

### 获取文章列表
```
GET /api/data/articles?page=1&per_page=20
```

### 情感分析
```
POST /api/sentiment/analyze
Body: {"text": "这个产品非常好"}
```

### 情感分布统计
```
GET /api/analysis/sentiment-distribution
```

### 热词统计
```
GET /api/analysis/hotwords?topK=50
```

### 情感趋势
```
GET /api/analysis/trend?days=7
```

## 🎓 毕业设计演示要点

1. **技术架构** - 前后端分离、RESTful API
2. **多平台采集** - 微博、抖音、B站、小红书
3. **情感分析** - 基于词典，可解释性强
4. **可视化展示** - ECharts图表，交互式界面
5. **实时更新** - 支持定时采集和手动更新
6. **低成本部署** - 无需GPU，使用MySQL

## ⚠️ 注意事项

1. **首次启动前端** - npm install需要几分钟
2. **端口占用** - 确保5000和8080端口未被占用
3. **MySQL服务** - 确保MySQL服务已启动
4. **数据更新** - 定期运行爬虫采集新数据

## 🔧 故障排除

### 前端无法访问
- 检查npm run serve是否成功启动
- 查看命令行是否有错误信息

### 后端连接失败
- 检查后端服务是否运行
- 访问 http://localhost:5000/api/data/articles 测试

### 数据库连接失败
- 确认MySQL服务已启动
- 检查用户名密码是否正确
